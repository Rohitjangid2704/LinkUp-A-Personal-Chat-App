# Auto-Analyst Data Processing Pipeline

This document explains exactly what happens when you upload a CSV or Excel file to the Auto-Analyst backend, and the different capabilities you can extract from your datasets.

## 1. File Upload & Ingestion
When you upload a dataset (e.g., `Housing.csv`), the file is read into memory (usually converting Excel to CSV or parsing it directly using Python's `pandas` or `polars`). 

Instead of sending the *entire* file to OpenAI (which would be slow, massive, and expensive), the backend acts as a **Code Execution Engine**. It reads the columns, statistics, and metadata of your file, and uses the AI to write Python code to analyze it!

## 2. The Two Main Workflows

Once your file is loaded, you have two core ways to interact with it:

### Workflow A: Interactive Chat Analysis
You can chat with your data just like ChatGPT. When you ask a question like *"Are house prices related to square footage?"*:
1. **The Planner Agent:** Determines which specialized AI agent is best equipped to handle your request.
2. **Code Generation:** The chosen agent generates exact Python code (`pandas`, `matplotlib`, `scikit-learn`) to answer your prompt.
3. **Execution Environment:** The backend securely executes that Python code locally on your uploaded file.
4. **Response:** The system returns exactly what the code output: 
   - A cleaned/modified CSV
   - A text summary or statistical result
   - A Base64-encoded image (e.g., a scatter plot or heatmap)

### Workflow B: "Deep Analysis" (Automated End-to-End Report)
If you just want a complete understanding of a new dataset without chatting, you can trigger a "Deep Analysis". This kicks off a massive, multi-step pipeline where the AI acts as an autonomous data scientist:

#### Phase 1: Data Preprocessing
*(Using the `preprocessing_agent`)*
- Detects and handles missing values (NaNs/Nulls).
- Identifies outliers in your numerical columns.
- Converts data types (e.g., fixing a date column that loaded as text).

#### Phase 2: Exploratory Data Analysis (EDA) & Statistics
*(Using the `statistical_analytics_agent`)*
- Calculates mean, median, standard deviation, and variance for numeric columns.
- Looks at categorical distributions (e.g., counts the frequency of different "Neighborhoods").
- Finds correlations (e.g., discovering that "Bedrooms" is highly correlated with "Price").
- Runs hypothesis testing if applicable (e.g., t-tests or ANOVA).

#### Phase 3: Data Visualization
*(Using the `data_viz_agent`)*
- Automatically generates the most appropriate charts based on the data types.
- **Histograms/Boxplots:** To show the distribution of prices.
- **Scatter Plots:** To show the relationship between variables (Sq. Ft vs Price).
- **Correlation Heatmaps:** To visualize the correlation matrix.

#### Phase 4: Feature Engineering
*(Using the `feature_engineering_agent`)*
- Creates new, useful columns based on existing ones. (e.g., combining "Beds" and "Baths" into a "Total Rooms" score, or creating a "Price per Sq Ft" column).
- Encodes categorical variables (One-Hot Encoding) so they are ready for Machine Learning.

#### Phase 5: Machine Learning Modeling
*(Using the `sk_learn_agent`)*
- Identifies if your dataset is better suited for **Regression** (predicting a continuous number, like House Price) or **Classification** (predicting a category).
- Automatically splits the data into Training and Testing sets.
- Trains baseline models (Linear Regression, Random Forest, etc.).
- Evaluates the model (outputting R-squared, RMSE, or Accuracy metrics).

#### Phase 6: Report Generation
- The backend takes all the text, statistics, charts, and model results generated by the 5 steps above and dynamically stitches them together into a beautiful, self-contained **HTML Report**. 
- You receive a downloadable HTML file containing the entire life cycle of your dataset's analysis!

---

## The AI Agents Under The Hood
The real magic is that the backend dynamically routes tasks to specialized agents (defined in `agents_config.json`). By keeping the prompts specific, the AI writes significantly better Python code:

1. **Preprocessing Agent:** Experts at `pandas` data cleaning.
2. **Statistical Analytics Agent:** Experts at `scipy` and statistical math.
3. **Data Visualization Agent:** Experts at `matplotlib`, `seaborn`, and `plotly`.
4. **Machine Learning Agent:** Experts at `scikit-learn` modeling and evaluation.
5. **Feature Engineering Agent:** Experts at data transformation and scaling.
6. **Polars Agent:** An alternative to Pandas optimized for lightning-fast analysis on massive, millions-of-rows datasets.

## Summary
In short, the backend doesn't just read your CSV. It reads the *schema* of your CSV, gives it to an AI, asks the AI to write a Python script to perform data science on that schema, securely runs that Python script on your actual file, and sends you back the final results (Charts, Clean Data, Models, HTML Reports).
